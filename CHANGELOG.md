# Changelog

## V2.6.0 - 2026-07-23（内置语音模型与安装目录数据）

- SenseVoice 量化语音模型改为随 DCSHUB 安装包内置，用户安装后即可离线使用，不再从 GitHub 下载或解压模型。
- 构建流程从国内 ModelScope 镜像准备模型，并对模型与词表执行文件大小和 SHA-256 双重校验，损坏文件不会进入安装包。
- 正式版的设置、API 配置、手册索引与运行数据统一存放到 `DCSHUB.exe` 同级的 `data` 目录；首次启动会从旧 `%APPDATA%\dcs-control-hub` 非覆盖迁移，旧数据保留作为回退。
- 安装器升级时会保留安装目录内的 `data`，避免覆盖安装丢失设置；开发模式使用源码目录下隔离的 `.tmp/dev-user-data`。
- 修复内置手册自定义按键被 DCSHUB 独占的问题：键盘钩子只旁听按下/松开事件并始终继续传递，短按、长按语音与前台程序原按键功能可同时工作。
- 非独占式按键监听进程异常退出后会自动重启，不再退回会抢占按键的系统全局快捷键注册。

## V2.6.0-local.1 - 2026-07-23（自由热键与轻量调试流程）

- 超级手册呼出键明确支持普通单键、组合键、数字键盘键、常用标点键和 Windows Game Controller 外设按钮，不再以“组合键”作为设置限制。
- 修复 `Super/Win` 键被主进程误解析为 `Ctrl`，以及部分标点键能够录制但无法保存的问题。
- 热键录制状态新增明确的取消按钮，`Esc` 也可以作为用户自定义单键。
- 新增热键解析集成测试，覆盖单键、组合键、Win 修饰键、数字键盘、标点键与外设按钮。
- 新增 `npm run dev:fast` 轻量调试入口，复用已有 Core/OpenXR 构建并启用 Renderer 热更新与 Electron 主进程自动重启，无需为每次小改动重新打包绿色版。

## V2.5.1 - 2026-07-23（VR 内置手册、语音输入与核心架构大版本）

- 原生 OpenXR VR 内置手册浮窗正式上线，支持视线方向呼出、世界空间固定、弧形拖动、独立尺寸记忆、问答与引用原页联动。
- 新增 SenseVoice 本地语音输入：短按呼出键显示/隐藏，长按录音，松开识别，提供 1.6 秒修改窗口后自动提问；支持键盘组合键和游戏控制器按钮。
- 软件框架重构为 React/Electron 交互层、独立 .NET 10 Core Host 与 Windows/OpenXR 原生组件分层架构，提升后台监控、退出清理和功能扩展的稳定性。
- 超级手册 RAG 库完成索引、机型隔离、术语语义、来源优先级、连续章节检索、证据审校、引用页预览和缓存体系重构。
- 优化本地检索与联网研究链路；按当前基准统计，本地平均搜索时间缩短约 60%，联网搜索时间缩短约 40%。
- API 供应商扩展到 DeepSeek、阿里云百炼千问和硅基流动；本地手册模型可自由选择，联网搜索使用供应商原生联网能力。
- 修复 TrackIR 与旧默认 F9 热键冲突、窗口关闭后后台进程残留、语音机型/武器术语误识别及弱相关手册误答等问题。
