# Changelog

本项目从 V1.6 开始通过 GitHub Releases 记录公开版本。

## V2.1.6 - 2026-07-22

- 公开发布改为仅提供按机器安装的 NSIS 安装包，不再构建或上传绿色版 ZIP；本地仍可使用 `npm run build:debug-portable` 生成免安装调试目录。
- 安装版与主程序统一请求管理员权限，保留 Electron 的 Per-Monitor DPI 适配，避免不同解压路径继承不一致的权限和缩放兼容设置。
- 修复首次安装默认不启用内置软件后，软件目录集成测试仍按旧默认值断言而导致的 GitHub CI 连续失败。
- Release 工作流同时接受 `V*` 与 `v*` 标签，并只为安装包生成和发布 SHA-256 校验值。

## V2.0 - 2026-07-20

- 超级手册日常问答固定使用 DeepSeek V4 Flash，不再向用户暴露模型选择；设置页仍可随时更换 API Key。
- 本地答案下方新增按需“在线搜索”，使用 DeepSeek V4 Pro、MAX 思考和原生联网搜索核对官方与一手资料，不点击时不会产生额外费用。
- 步骤答案改为“逐字证据账本 + 新手教学改写”：原文只负责证明事实，用户看到的是通俗的操作说明和作用解释，同时严格保留专业术语、控制项、数值与系统角色。
- 操作类回答必须逐条通过本地证据子串、引用编号、技术词和术语角色校验；无法形成可靠操作链时拒绝猜测，不再缓存失败答案。
- 检索会补齐同一章节的连续步骤页，并新增跨重启的已验证答案缓存；相同问题在手册未变化时可直接复用稳定结果。
- 多来源检索新增权威性与时效性分层：当前 DCS 客户端官方手册 > 其他官方副本 > Chuck 社区手册 > 用户资料；同级优先明确版本更新者，只有整层证据不足才降级，禁止拼接互相冲突的新旧流程。
- 放宽过度机械的短引文术语门禁：操作仍必须有逐字原文，专业术语可由同一引用页佐证；既允许面向新手解释，又不能借用未引用页面或低优先级资料补步骤。
- 在线来源通过受限 HTTPS 系统浏览器通道打开，避免链接在无边框 Electron 窗口内误导航。

## V1.8.12 - 2026-07-20

- 移除“头盔 + 标记目标”按精确措辞触发的全机型硬编码规则，改为稳定任务族；增删“目标”“操作”等普通词不再切换检索逻辑。
- 检索从限定机型手册中确定真实流程与术语，不再把 F-16 的 MARKPOINT/SPI 规则套到 F/A-18C；Hornet 现在命中 JHMCS 对地指定正文。
- 建立控制器、控制权、目标状态和可保存点的术语角色约束：TDC/TMS/DMS、TDC priority/SOI、SPI/TGT designation、MARKPOINT 不得互相改名。
- 任务证据门槛会剔除校准页、更新记录、目录和 UFC/TIME 等偶然命中页；来源不足时不再让单张弱相关页面进入回答。
- 审校输出改为删除无来源操作行并保留合格答案，修复 Markdown 标题、引用换行导致整份正确答案被误拒的问题。
- 增加 F-16、F/A-18C、C-130J 的同义句、增删词、恶意误排和跨机型术语隔离回归矩阵。

## V1.8.11 - 2026-07-20

- 修复“F-16 用头盔标记目标”等问题被 HMCS 开机/校准章节压过核心操作正文的问题；检索新增核心任务术语、跨页步骤锚点和任务优先重排。
- 明确拆分 MARKPOINT、武器目标指定与空中目标锁定，用户未说明武器/子模式时不再生成一套虚假的通用流程，也不会混入其他机型的控制逻辑。
- 步骤类答案增加第二阶段证据审校：按键、开关、模式、数值和顺序必须有同一行有效来源，无法通过审校时拒绝输出可疑步骤。
- 修复 DeepSeek 将 `[S2]` 引用跨行输出时被误判为无来源步骤的问题，并加入恶意重排与无来源按键幻觉回归测试。

## V1.8.10 - 2026-07-20

- 修复专注问答模式中手册图片放大弹窗被专注卡片遮挡的问题；图片预览现在使用独立的最高层级。
- 图片预览打开时按 Esc 只关闭大图并保留专注模式，再次按 Esc 才退出专注模式。

## V1.8.9 - 2026-07-20

- 全面重构超级手册 V5 检索流水线，修复机型名重复参与正文评分、使同机型所有页面虚高的问题；机型现在只用于严格限定文档，任务查询只匹配正文。
- 查询融合加入本地 DCS/军航术语、DeepSeek 标准术语、任务意图和多条互补查询的分级权重；即使语义路由失败，常见任务仍有确定性本地召回兜底。
- 重新设计章节与邻页扩展权重：目录只用于定位正文，邻页仅作为低权重上下文，避免一张误命中页把整组无关页面推到结果顶部。
- 重排前增加正文证据密度、流程动作、参考页识别和子任务覆盖评分；DeepSeek 无法再用只偶然出现关键词的页面替换高相关正文。
- 回答上下文会围绕实际命中术语截取，不再固定发送片段开头；同一问题的语义与重排缓存包含候选签名和检索版本，避免旧结果污染新算法。
- 增加 C-130J 空投、恶意误选应急设备页、中文提问英文手册、跨机型隔离、目录章节追踪和重复提问稳定性回归测试。

## V1.8.8 - 2026-07-20

- 修复超级手册专注模式层级低于侧边栏的问题，现在专注界面会完整覆盖工作区与侧边栏。
- 提问输入框支持 Enter 直接提交、Shift+Enter 换行，并避免中文输入法选词时误提交。
- 移除“可检索页面”和“检索片段”统计卡；已索引手册、本地索引缓存与已入库手册改为三个同尺寸入口。
- 已入库手册改为摘要卡片，点击后以大窗口显示来源分类和完整手册列表，使默认问答区域使用全部可用宽度。

## V1.8.7 - 2026-07-20

- 点击提问输入区即进入覆盖工作区的专注问答模式，回答、引用和相关图片可以使用主界面的最大空间，并支持按 Esc 或按钮退出。
- 引用页预览会在生成时自动裁掉 PDF 外围空白，并在对应回答段落右侧显示；点击可放大查看。
- 引用来源和预览大图中的“定位到原手册”会使用内置 PDF 窗口直接打开对应页码。

## V1.8.6 - 2026-07-20

## V1.8.5 - 2026-07-20

- DeepSeek 意图分析现在结构化输出机型、系统、用户任务、标准术语与查询；结果必须与本地机型目录核对后才能限定来源。

- 资料库缺少用户指定机型时直接提示添加相应手册，不再使用其他机型资料代替；支持多机型比较路由。
- 新增中文阿帕奇、动态未知新机型、缺失机型拒绝回答和跨机型隔离回归测试。
- 启动后从既有解析缓存升级 V4 索引，以便动态机型识别同时覆盖已经入库但旧版未识别的模块，无需重新解析 PDF。
