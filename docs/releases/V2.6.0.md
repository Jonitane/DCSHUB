# DCSHUB V2.6.0 更新说明

<!-- dcshub-push-update -->

V2.6.0 解决语音模型在部分网络环境下无法下载的问题，并修复内置手册呼出键占用前台程序按键的严重兼容问题。本版本作为面向所有已开启更新检查用户的建议更新发布。

## 更新重点

- SenseVoice 量化语音模型已随安装包内置，安装完成即可离线语音识别，不再要求用户自行访问 GitHub 下载或解压模型。
- 超级手册呼出键可自由设置为普通单键、组合键、数字键盘键、常用标点键或 Windows 游戏控制器按钮。
- 键盘呼出键改为非独占监听：DCSHUB 可以识别短按和长按，DCS、TrackIR 及其他前台程序仍会正常收到同一个按键。
- 设置、API 配置、手册索引和运行数据统一放到 `DCSHUB.exe` 同级的 `data` 目录，减少系统盘用户目录权限或访问异常带来的问题。

## 语音模型与使用方法

SenseVoice 的 `model.int8.onnx` 和 `tokens.txt` 现在直接包含在 Windows 安装程序中。打包前会从国内 ModelScope 镜像准备文件，并执行固定大小和 SHA-256 校验；文件损坏时构建会直接失败，不会发布不完整模型。

安装后使用：

1. 打开“设置 → 超级手册 → 内置手册窗口”，选择麦克风并设置呼出键。
2. 短按呼出键显示或隐藏手册窗口。
3. 长按呼出键说话，松开后系统补录短暂尾音并在本地识别。
4. 识别文本进入输入框后保留 1.6 秒修改时间；按 `Enter` 可立即发送，开始编辑会取消自动提交。
5. 需要联网资料时，可按 `Ctrl+Enter` 或点击联网搜索。

模型只在本机运行，录音不会作为整段音频上传。语音后处理继续与超级手册术语库共用机型、武器、传感器和航电缩写，可校正 `iPhone → F-14`、“节达姆/杰达姆”→ `JDAM` 等常见误识别。

## 呼出键不再占用前台程序

此前底层键盘钩子在识别到绑定键后会截断系统按键链，导致软件运行期间 DCS 或其他程序无法继续使用该键。V2.6.0 已改为只旁听按下/松开事件并始终传递原始输入：

- 短按显示/隐藏和长按语音均保持可用。
- 原按键仍会正常传给当前前台程序。
- 监听进程异常退出时会自动重启。
- 不再降级到会独占按键的系统全局快捷键注册。

## 用户数据迁移

- 正式版和打包绿色版使用 `<安装目录>\data`。
- 从旧版本首次启动时，会从 `%APPDATA%\dcs-control-hub` 复制已有配置、手册索引和运行数据。
- 迁移采用非覆盖方式，不会覆盖安装目录中已经存在的新设置。
- 旧目录不会被删除，可在必要时用于回退。
- 旧版单独下载的语音模型和 Chromium 临时缓存不会重复复制。
- 覆盖安装升级会保留安装目录中的 `data`。

> 正常主动卸载会删除整个 DCSHUB 安装目录。如需保留设置，请在卸载前备份 `data` 文件夹。

## 今日问题修复汇总

- 修复用户网络无法访问 GitHub时，SenseVoice 下载或压缩包解压报 `tar 1` 的问题；模型改为随包内置。
- 修复自定义手册呼出键被 DCSHUB 独占、软件运行期间其他程序无法使用该键的问题。
- 修复旧默认 `F9` 与 TrackIR 热键冲突；旧设置会迁移到 `Ctrl+Alt+M`，关闭手册功能会停止监听。
- 修复长按语音可能被误判为短按并关闭界面的问题，松手后保留尾音和文字确认时间。
- 修复主窗口退出后 Electron、Core Host 或 VR Bridge 可能残留的问题。
- 修复桌面和 VR 内置手册不能分别记住用户调整尺寸的问题。
- 修复右侧引用图片不能随答案滚动窗口实时保持可见的问题。
- 修复 F-14、JDAM 及多种机型、武器、传感器和航电术语的语音误识别。
- 修复弱相关页面可能生成错误答案；无意义或过于抽象的问题会明确提示使用联网搜索。
- 修复旧答案中的“成功判断”被错误转换成额外编号步骤的问题。
- 超级手册呼出键不再限制为组合键，支持自由录制键盘键和游戏控制器按钮。

## 升级说明

- 可以直接覆盖安装升级到 V2.6.0。
- 本版本内置约 228 MB 的量化语音模型，因此安装程序和安装体积会明显增加。
- 更新提示只负责通知并打开官方 GitHub Release 页面，不会后台自动下载安装。
- 当前安装程序尚未进行商业代码签名，Windows 可能显示未知发布者提示。
- OpenXR 原生组件有更新时，请彻底退出并重新启动 DCS 后再测试 VR 浮窗。
